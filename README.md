# GuardPulse Connect

> **Production-Grade Agentic RegTech Framework & Onboarding Marketplace**

GuardPulse Connect is an enterprise-ready, **Agentic Retrieval-Augmented Generation (RAG)** platform designed to automate complex legal and technical compliance audits for B2B startup onboarding. Moving away from fragile, single-prompt LLM wrapper architectures, GuardPulse utilizes a deterministic, event-driven **multi-agent AI swarm** running over specialized vector stores to perform multi-dimensional risk surface analysis.

## ⚡ Core Tech Stack & Engineering Choices

* **Asynchronous API Gateway:** Built with **FastAPI** using `python-multipart` to stream high-throughput binary files into processing nodes via dedicated REST webhooks.
* **Hardware-Aware Containerization:** Custom multi-stage `Dockerfiles` configured to pull **CPU-only PyTorch mirrors** (`--index-url [https://download.pytorch.org/whl/cpu](https://download.pytorch.org/whl/cpu)`), trimming down image footprint by eliminating gigabytes of unutilized CUDA libraries for optimized deployment.
* **Deterministic Orchestration:** Avoided high-overhead agent frameworks (e.g., CrewAI/LangChain) in favor of a **pure Python orchestration layer**. State management, message formatting, and agent communication paths remain fully transparent and deterministic.
* **Strict Type-Safe Validation:** Powering agent-to-agent data flows entirely with **Pydantic v2** models, ensuring that unstructured LLM payloads strictly align with legal and risk schemas before moving down the pipeline.
* **High-Speed Inference Engine:** Powered by the **Groq SDK** executing **Llama 3.3 70B Versatile**, dropping full-swarm consensus processing down to ~2 minutes (a $20\times$ improvement over local CPU-bound models).

## 🤖 The Multi-Agent Swarm

Instead of a broad, singular sweep, the workload is distributed across three isolated specialist roles synchronized by an Orchestrator core:

1. **Auditor Agent (Contextual RAG):** Queries a localized **ChromaDB** instance packed with statutory clauses from the **DPDP Act 2023 (India)** and the **EU AI Act 2024** (split via dynamic section boundaries rather than generic word counts). Evaluates files across specific regulatory control vectors.
2. **Technical Architect Agent:** Analyzes the file for technical debt, PII exposure, weak authentication structures, and encryption gaps.
3. **Critic Agent (Anti-Hallucination Guard):** Cross-checks every legal citation generated by the Auditor against original source laws using semantic verification. Flags claims failing a strict confidence threshold.

## 📊 Evaluation & Mathematical Scoring Matrix

The final composite score balances legal posture, technical hygiene, and factual reliability:

$$GuardPulse\ Score = (Legal\ Score \times 0.50) + (Tech\ Score \times 0.35) + (Critic\ Bonus \times 0.15)$$

$$\text{Where: } Critic\ Bonus = 100 \times (1 - Hallucination\ Rate)$$

> [!CAUTION]
> **Deterministic Hard-Cap:** The discovery of any **CRITICAL** vulnerability by the Technical Agent overrides the matrix formula entirely, forcing a hard ceiling score of **55** and instantly assigning a `NOT_READY` badge state.

| Score Threshold | Assigned Badge | Automated Routing Path |
| --- | --- | --- |
| **80 – 100** | `ENTERPRISE_READY` | Issue security certification; route directly to Slack alerts and Notion Enterprise CRM. |
| **60 – 79** | `CONDITIONAL` | Generate targeted remediation instructions; return to the founder via mail. |
| **Below 60** | `NOT_READY` | Reject entry into the marketplace registry; open manual review window. |

## ⚙️ Automated Integration Architecture (n8n Layer)

The infrastructure couples raw AI computation with a production-grade automated pipeline via **n8n**:

* **Binary Data Pipeline:** Handled directly outside n8n's binary stream using the standard form field array to bypass structural JSON encoding limitations on raw files.
* **Secured Notification Mesh:** Leverages production-level Google Cloud Console OAuth2 workflows to securely distribute customized scorecards to target recipients.
* **CRM Data Sync:** Automatically updates and builds enterprise tables inside **Notion**, mapping out properties like *Score, Badge, Legal/Tech Subscores, Submitted At,* and *Report Tracking URLs* in real-time.
